🧠 Unit 1: OS Basics

🔍 What is Operating System?

An OS is system software that acts as a bridge between user and hardware. It manages CPU, memory, I/O devices and ensures execution of programs.

"OS ek bridge hai jo user aur hardware ke beech coordination banata hai."
User
 ↓
Operating System
 ↓ ↓ ↓
CPU   Memory   I/O Devices

❓ Quiz:

  • Q1: Main role of OS? ✅ B. Managing hardware resources
  • Q2: OS bridges between? ✅ B. User and hardware

Shortcut Tag: OS = Resource Manager + Interface Layer

📂 Types of Operating Systems

Different types of OS are used based on task handling, time sensitivity, and resource management.

  • Batch OS: Executes grouped jobs without interaction.
  • Time-Sharing OS: CPU slices are shared among users.
  • Real-Time OS: Used in robotics, medical devices.
  • Distributed OS: Controls multiple systems as one.
  • Embedded OS: Lightweight for appliances.

Shortcut Tag: Task Style + User Load + Response Time

📝 Quick Quiz:

  • Q: OS used in washing machines? ✅ Embedded OS
  • Q: OS that gives CPU slices? ✅ Time-Sharing OS

🧩 System Calls in Operating Systems

System Calls OS ke core interface hote hain — jinke through user programs OS services ko access karte hain.

📘 Definition:

System Call is a programmatic way in which a computer program requests a service from the operating system.

📑 Types of System Calls:

Type Description
Process ControlCreate, terminate, load, execute processes
File ManagementOpen, read, write, close files
Device ManagementRequest, release, attach devices
Information MaintenanceGet system time, date, process info
CommunicationSend/receive messages between processes

🧪 Example (C Code):

#include <unistd.h>

int main() {
  write(1, "Hello Gaurav!\n", 14);
  return 0;
}

write() yahan ek system call hai jo output screen pe data bhejta hai.

🔄 Process Management in Operating Systems

OS processes ka management karta hai — creation, execution, scheduling, termination sab yahi handle karta hai.

🔢 Classification:

  • Process: Ek executing program jiska apna state, memory, aur resources hote hain.
  • Process States: New → Ready → Running → Waiting → Terminated
  • PCB (Process Control Block): Process ka pura data store karta hai — PID, state, registers, memory info.
  • Context Switching: Ek process se dusre pe switch karne ka mechanism — CPU state save/restore hota hai.
  • Scheduling Types:
    • Long-term (job selection)
    • Short-term (ready queue se CPU allocation)
    • Medium-term (suspend/resume)

📎 Examples:

  • PCB: PID = 102, State = Running, Memory = 4MB
  • Context Switching: Performed by UNIX Kernel
  • Scheduling: Round Robin, FCFS, SJF

Shortcut Tag: Process Management = Process + States + PCB + Context Switch + Scheduling

  • Q3: Short-term scheduler ka kaam kya hai? ✅ B) CPU ko ready process dena

🛠️ Operating System Services

OS kuch essential services provide karta hai jo user aur applications ke liye environment ko efficient aur secure banate hain.

  • Program Execution: Programs ko load aur execute karta hai.
  • I/O Operations: Input/output devices ke saath interaction manage karta hai.
  • File System Manipulation: Files ko create, delete, read, write karta hai.
  • Communication: Inter-process communication ke liye mechanisms deta hai.
  • Error Detection: Errors detect karta hai aur fix karne ke actions leta hai.
  • Resource Allocation: CPU, memory, I/O devices jaise resources allocate karta hai.
  • Accounting: Resource usage ka track rakhta hai (monitoring/billing ke liye).
